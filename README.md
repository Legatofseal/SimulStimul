# Amoebas

Трёхмерный браузерный прототип симулятора жизни и эволюции. Игра работает без бэкенда и этапа сборки.

Демо: https://legatofseal.github.io/SimulStimul/

## Возможности прототипа

- конструктор организмов с очками стоимости и готовыми шаблонами;
- наследуемые характеристики, цвет покрова и мутации до ±10%;
- многоуровневый интеллект: более умные организмы учитывают больше факторов риска и выгоды;
- маскировка в биомах и способности «Хамелеон», «Рывок», «Регенерация» и «Яд»;
- понятный журнал причин действий, инспектор клеток и организмов;
- цели «контроль объёма» и «выживание года», экран результата и новый эксперимент;
- настоящее трёхмерное пространство аквариума: глубина, свободная камера и выбор объектов лучом;
- процедурные 3D-организмы: амёбоидные пришельцы, ресничные травоядные и бронированные древние хищники;
- плавная деформация тел, реснички, жвала, следы рывка, регенерация, наследуемая окраска и визуальная маскировка;
- объёмные биомы дна, растения, камни, бактериальные маты, поверхность воды, пузырьки и световые лучи;
- трёхмерный ИИ: организмы ищут пищу, свет, добычу и безопасный слой воды;
- локальный терморазряд, который уничтожает донную органику, распространяется и остывает под дождём;
- локальное сохранение и развитие мира во время отсутствия игрока.

Трёхмерная сцена создаётся процедурно через Three.js. Локальная копия Three.js и OrbitControls лежит в `vendor/three`; лицензия MIT сохранена рядом.

## Управление

- перетаскивание мышью или одним пальцем — вращение камеры;
- колесо мыши или жест двумя пальцами — масштаб;
- режим существа: тап по дну задаёт курс;
- «Нырнуть» / `E` / `Page Down` — уйти глубже;
- «Всплыть» / `Q` / `Page Up` — подняться;
- стрелки — ручное движение выбранного пришельца;
- Shift+клик в режиме бога — создать терморазряд;
- «Общий вид» — вернуть исходную камеру.

## Локальный запуск

Из-за ES-модулей игру нужно открывать через статический HTTP-сервер из корня репозитория:

```powershell
npx serve .
```

После запуска откройте адрес, показанный сервером.

## GitHub Pages

В репозитории уже есть workflow `.github/workflows/pages.yml`.

1. Создайте GitHub-репозиторий и отправьте в него ветку `main`.
2. Откройте `Settings → Pages` и выберите публикацию через GitHub Actions.
3. После push workflow опубликует сайт на GitHub Pages.

## Azure Static Web Apps

Создайте ресурс **Static Web App** в Azure и подключите GitHub-репозиторий:

- App location: `/`
- API location: оставить пустым
- Output location: оставить пустым

Файл `staticwebapp.config.json` уже лежит в корне и задаёт fallback на `index.html`.

## Данные игры

Состояние сохраняется локально в `localStorage` браузера. Серверные аккаунты, общие миры и синхронизация между устройствами пока не используются.
